# TaskFlow API

REST-сервис для управления задачами с автоматическим назначением исполнителей.

## Технологии

- Python 3.11+
- FastAPI
- SQLAlchemy 2.0 + SQLite
- JWT-аутентификация (python-jose + passlib/bcrypt)
- Pytest + TestClient

## Структура базы данных

| Таблица       | Описание                                              |
|---------------|-------------------------------------------------------|
| `users`       | Сотрудники с уровнем квалификации (1–5) и ролью       |
| `tasks`       | Задачи с приоритетом, статусом и требуемой квалификацией |
| `assignments` | История автоматических назначений задач               |

## Алгоритм автоматического назначения

Эндпоинт `POST /tasks/{id}/auto-assign` выбирает оптимального исполнителя:

1. Отбирает активных сотрудников с `qualification_level >= required_qualification`
2. Вычисляет текущую загруженность каждого — сумма весов активных задач:
   - `low = 1.0`, `medium = 1.5`, `high = 2.0`, `critical = 3.0`
3. Рассчитывает итоговый балл:

$$score = \frac{workload}{qualification\_level - required\_qualification + 1}$$

4. Назначает сотрудника с наименьшим баллом

Чем выше загруженность и чем ниже превышение квалификации — тем хуже кандидат.

## Установка и запуск

```bash
cd "TaskFlow API"
pip install -r requirements.txt
uvicorn app.main:app --reload
```

Swagger UI: http://127.0.0.1:8000/docs

## Тесты

```bash
pytest tests/ -v
```

## API

### Аутентификация

| Метод | Путь             | Описание                       |
|-------|-----------------|-------------------------------|
| POST  | /auth/register  | Регистрация нового пользователя |
| POST  | /auth/login     | Вход, возвращает JWT-токен      |

### Пользователи (требуется Bearer-токен)

| Метод  | Путь            | Описание                          |
|--------|----------------|----------------------------------|
| GET    | /users/         | Список всех пользователей         |
| GET    | /users/me       | Текущий пользователь              |
| GET    | /users/{id}     | Пользователь по ID                |
| PUT    | /users/{id}     | Обновление профиля                |
| DELETE | /users/{id}     | Удаление (только роль admin)      |

### Задачи (требуется Bearer-токен)

| Метод  | Путь                         | Описание                              |
|--------|------------------------------|--------------------------------------|
| POST   | /tasks/                      | Создать задачу                        |
| GET    | /tasks/                      | Список задач (фильтры: status, priority) |
| GET    | /tasks/{id}                  | Задача по ID                          |
| PUT    | /tasks/{id}                  | Обновить задачу                       |
| DELETE | /tasks/{id}                  | Удалить задачу                        |
| POST   | /tasks/{id}/auto-assign      | Автоматически назначить исполнителя   |

### Назначения (требуется Bearer-токен)

| Метод | Путь                 | Описание                  |
|-------|---------------------|--------------------------|
| GET   | /assignments/        | История назначений        |
| GET   | /assignments/{id}    | Назначение по ID          |

## Роли пользователей

- `employee` — может создавать задачи и видеть данные
- `admin` — дополнительно может удалять любых пользователей

## Уровни квалификации

Значения от 1 до 5, где 5 — наивысший уровень.
